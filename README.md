# ErsatzTV HLS Copy 预处理工具（最终定版参数）

这是一个**独立可运行**的预处理程序目录，用于把源视频批量整理成更适合 ErsatzTV HLS `copy` 模式播放的文件。

## 适用目标

你的目标不是“播放时实时转码更强”，而是：

- 先离线预处理一次
- 播放时继续使用 `-c:v copy -c:a copy`
- 在低 CPU 环境下尽量稳定播放

---

## 这套工具固定使用的参数

### 视频

- 编码器：`libx264`
- `preset=slow`
- `crf=20`
- 分辨率策略：1080p 封顶（超过 1920x1080 则等比缩放，不超过则保持原分辨率，仅修正为偶数尺寸）
- 保持原始 fps，但转为 CFR
- GOP：1 秒
- 每 1 秒强制关键帧
- `-sc_threshold 0`
- `-bf 0`
- `-pix_fmt yuv420p`
- `setsar=1`
- `-profile:v high`
- level：自动（由 x264 根据分辨率和帧率选择，避免硬编码导致 4K/高帧率超限）
- `-avoid_negative_ts make_zero`

### 音频

- 编码器：`aac`
- 码率：`160k`
- 采样率：`48000`
- 声道：`2`

### 容器

- 输出：`mp4`
- `-movflags +faststart`

## 依赖

### 必需

1. **Python 3**
   - 建议：Python 3.10 或更高
   - 安装时建议勾选：`Add python.exe to PATH`
2. **ffmpeg.exe**
3. **ffprobe.exe**

- 不需要 `pip install`
- 不需要额外配置 ErsatzTV

本工具只依赖 **Python 标准库 + bin 目录中的 ffmpeg/ffprobe**。

## 目录结构

程序目录建议长这样：

```text
ErsatzTV-HLS-Copy-Prep/
├─ start_hls_copy_prep.bat
├─ hls_copy_prep_final.py
├─ README-使用说明.md
├─ python-3.14.3-amd64.exe   （可选，本地 Python 安装包）
├─ bin/
│  ├─ ffmpeg.exe
│  └─ ffprobe.exe
├─ source/
├─ target/
├─ done/
└─ logs/
```

各目录作用：

- `source/`：放待处理视频
- `target/`：放转码后的输出文件
- `done/`：成功处理后，原始文件会被移动到这里
- `logs/`：保存每个文件的日志、每次任务的汇总日志、预检查报告
- `bin/`：放 `ffmpeg.exe` / `ffprobe.exe`

## 使用方法

### 最简单的用法

1. 如果机器还没有 Python 3：
   - 把 `python-3.14.3-amd64.exe` 放在根目录
   - 双击 bat 后选 **[8] 安装 Python 3**
2. 把待处理视频放进 `source/`
3. 双击运行：

```text
start_hls_copy_prep.bat
```

4. 在菜单里选择你要的功能

## 预检查模式

这是**只扫描、不转码**的模式，适合你在正式处理前先摸清 source 里的情况。

### 它会做什么

- 扫描源目录中的视频文件
- 统计：
  - 文件数量
  - 总大小
  - 总时长
- 给出一个**经验性预计处理时间**
- 分析每个文件的基础风险特征
- 列出**最值得优先关注的文件**
- 在 `logs/` 中生成报告

### 它关注哪些风险点

例如：

- 视频不是 H.264
- 音频不是 AAC
- 容器不是 MP4
- SAR 不是 1:1
- 分辨率存在奇数边
- 帧率信息不规整，疑似 VFR
- 分辨率过高 / 帧率过高

### 生成哪些报告

会在 `logs/` 中写出：

- `scan-YYYYMMDD-HHMMSS-report.txt`
- `scan-YYYYMMDD-HHMMSS-report.json`

你可以先看这个报告，再决定什么时候开正式转码。

---

## 正式预处理模式

进入正式模式后，它会：

1. 让你确认或输入：
   - 源目录
   - 输出目录
   - done 目录
   - logs 目录
   - bin 目录
2. 扫描源文件
3. 统计：
   - 文件数量
   - 总大小
   - 总时长
4. 给出一个**经验性预计处理时间**
5. 列出前几个待处理文件
6. 给出前几个优先关注文件
7. 让你确认是否开始
8. 开始逐个处理并实时输出当前文件进度
9. 成功后：
   - 新文件写到 `target/`
   - 原文件移动到 `done/`
   - 日志写到 `logs/`

### 防重复转码保护（新增）

这次还加了一层防呆逻辑：

- 如果发现 `target` 里已经存在**完整有效**的目标文件
- 但 `source` 里原文件还在

那么工具会：

- **不再重复转码**
- 直接把 `source` 里的原文件移动到 `done`
- 在日志里记录这是一次“修复历史残留”

这样可以避免因为上次异常中断或旧版脚本遗留，导致重复触发重复转码。

## 日志说明

### 单文件日志

每处理一个文件，会生成一个 `.log` 文件，里面包含：

- 源文件路径
- 输出文件路径
- 实际 ffmpeg 命令
- ffmpeg 运行输出
- 过程中的进度信息

### 本次任务汇总日志

每次正式运行还会生成：

- `run-YYYYMMDD-HHMMSS.jsonl`
- `run-YYYYMMDD-HHMMSS-summary.json`

### 预检查报告

每次扫描模式会生成：

- `scan-YYYYMMDD-HHMMSS-report.txt`
- `scan-YYYYMMDD-HHMMSS-report.json`

---

## 处理结果规则

### 成功

- 输出文件进入 `target/`
- 原始文件移动到 `done/`

### 如果目标文件已存在且有效

- 不重复转码
- 直接把原始文件移动到 `done/`
- 以“已修复历史残留”的方式记录到日志和 summary

### 失败

- 原始文件会留在原来的 `source/`
- 失败原因写入日志

## 为什么这套参数适合你的场景

你的核心目标是：

- ErsatzTV 最终播放时必须继续走 `copy`
- 不依赖运行时高 CPU 转码

所以关键不在于“播放时转得更猛”，而在于：

> **提前把源文件整理成更适合 HLS copy 分片的结构**。

这套固定参数的目的就是：

- 保留较好的画质
- 获得规则关键帧
- 让 HLS 分段更稳定
- 降低卡顿 / 回跳 / 重播已播放画面的概率

---

## 你后面如何接入 ErsatzTV

建议把 `target/` 中的处理后文件，作为新的媒体来源导入到 ErsatzTV 测试库 / 正式库。

ErsatzTV 侧继续保持：

- HLS Segmenter
- `-c:v copy`
- `-c:a copy`

也就是：

- **离线预处理时转码一次**
- **在线播放时继续 copy**

## 常见问题

### 1）是不是必须安装 Python 3？

**是的，当前这套工具需要 Python 3。**

但现在已经支持把 Python 安装包直接放到根目录，然后从菜单里启动安装。

### 2）还需要别的依赖吗？

**不需要。**

只要：

- Python 3
- `bin/ffmpeg.exe`
- `bin/ffprobe.exe`

就够了。

### 3）为什么不直接做成纯 bat？

因为：

- 批量探测媒体信息
- 统计总时长
- 预估耗时
- 风险分析
- 做进度输出
- 生成结构化日志
- 做转码结果校验和防重复保护

这些用 Python 做更稳，也更容易维护。

bat 在这里主要负责：

- 启动
- 环境检查
- 菜单化入口
- 引导安装 Python
- 更友好的交互体验

---

## 建议的使用习惯

1. 先运行一次**预检查模式**
2. 看 `logs/` 里的 `scan-report`
3. 确认目录和文件都没问题后，再运行正式预处理
4. 处理后的文件进入 `target/`
5. 确认 `source` 中对应文件已被移走，且 `done/` 中已经归档
6. 保留 `done/` 一段时间，不要立刻删除原文件

---

## 一句话总结

这不是“播放时硬转码”，而是：

> **先把文件整理好，之后让 ErsatzTV 播放时继续 copy。**
